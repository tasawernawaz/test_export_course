Consider the continuous-time, discrete-state Markov process above. In this figure, the machine is in:

        • State 1 when it is up and making good parts (Up, Good);
        • State 2 when it is up and making bad parts (Up, Bad);
        • State 3 when it is down after an operational failure (Down, Operational);
        • State 4 when it is down after the quality failure has been detected (Down, Quality);

All transition times are assumed to be exponentially distributed with rates indicated.

The transition from state 1 to state 2 is a quality failure. The transition from states 1 or 2 to state 3 are operational failures.

Quality failures are not observed. As soon as the machine is determined to be in state 2 (ie, when the quality failure is detected), a quality repair begins and the machine is considered to be in state 4. However, while the machine is in state 2, an operational failure may occur before the machine is found to be in that state. In that case, it goes to state 3 instead of state 4.

After the quality repair, the machine returns to state 1. The mean time the machine spends in state 4 is \(1/m\).

Operational failures are observed as soon as they occur and the repair begins immediately. Then mean time to repair is \(1/r\) and the machine returns to state 1. Since the people observing the machine don't know whether it is in state 1 or state 2 before operational failure, the repair includes correcting the visible problem that stated the repair and the quality repair that may or may not be necessary. Consequently, the mean time the system is in state 3 is longer than the mean time it is in state 4 (ie, \( 1/r>1/m\)).

Assume that the production rate while the machine is in states 1 or 2 is \(\mu \) parts per time unit.

Assume \(\mu=1, r=0.05, m=0.07, q=0.01, p=0.005, d=0.1\) as base values.
